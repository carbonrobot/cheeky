Configuration

Configure your angular application by defining the keys and endpoints

angular.module('app', ['ngSanitize', 'cheeky'])
    .config(function(cheekyCMSProvider){
        cheekyCMSProvider.map({
            "website1": "/path/to/cheeky-cms/website1",
            "website2": "/path/to/cheeky-cms/website2"
        });
    });

Keys map to the first segment of a content path. If the content key in the CMS for a particular piece of content is website1.aboutpage.title, the key is website1.

The first time the application encounters the key website1, it will download and cache all content under that path so further calls to that same key will not make a call to the REST endpoint.

Getting content with angular.js

Map content to a CMS key using a directive as an element

// using an element
<cheeky-content path="content.msgs.welcome"></cheeky-content>

// using an attribute
<div cheeky-content="content.msgs.welcome"></div>

Enable content editing with angular.js

Content can be made editable by including the following script tag and adding the cheeky-editable attribute.

<script src="public/js/cheeky-editor.min.js"></script>
// using an element
<cheeky-content path="content.msgs.welcome" cheeky-editable></cheeky-content>

// using an attribute
<div cheeky-content="content.msgs.welcome" cheeky-editable></div>

Enabling / Disabling editing can be done by removing the script based on user rights or by omitting the script in production environments (in multi-environment sites).
